## 1. 입력과 출력

### BufferedReader

필요한 import:

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.IOException;
```

역할:

- 표준 입력을 빠르게 읽는다.
- 한 줄 전체를 문자열로 읽는다.

주요 메서드:

- `readLine()`

반환값:

- `String`
- 입력이 끝나면 `null`

예시:

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
String line = br.readLine();
System.out.println(line);
```

입력:

```text
hello world
```

출력:

```text
hello world
```

주의:

- `readLine()`은 문자열 전체를 읽는다.
- 숫자로 쓰려면 직접 형변환해야 한다.
- `IOException` 처리가 필요하다.

### StringTokenizer

필요한 import:

```java
import java.util.StringTokenizer;
```

역할:

- 한 줄을 공백 또는 지정 구분자로 잘라 토큰 단위로 읽는다.

주요 생성자:

- `new StringTokenizer(String s)`
- `new StringTokenizer(String s, String delim)`

주요 메서드:

- `nextToken()`
- `hasMoreTokens()`
- `countTokens()`

반환값:

- `nextToken()`: `String`
- `hasMoreTokens()`: `boolean`
- `countTokens()`: `int`

예시:

```java
StringTokenizer st = new StringTokenizer("10 20 30");
System.out.println(st.nextToken());
System.out.println(st.nextToken());
System.out.println(st.nextToken());
```

출력:

```text
10
20
30
```

코딩테스트 입력 예시:

```java
StringTokenizer st = new StringTokenizer(br.readLine());
int n = Integer.parseInt(st.nextToken());
int m = Integer.parseInt(st.nextToken());
```

주의:

- 토큰이 없는데 `nextToken()`을 호출하면 예외가 난다.
- 공백 여러 개가 있어도 기본적으로 잘 처리된다.

### 숫자 변환

필요한 import:

- 없음

주요 메서드:

- `Integer.parseInt(String s)`
- `Long.parseLong(String s)`
- `Double.parseDouble(String s)`

반환값:

- `int`
- `long`
- `double`

예시:

```java
int a = Integer.parseInt("123");
long b = Long.parseLong("10000000000");
double c = Double.parseDouble("3.14");

System.out.println(a);
System.out.println(b);
System.out.println(c);
```

출력:

```text
123
10000000000
3.14
```

주의:

- 범위를 넘는 숫자를 `parseInt`로 읽으면 예외가 난다.
- 값 범위가 크면 처음부터 `long`을 고려한다.

### StringBuilder

필요한 import:

- 없음

역할:

- 문자열을 효율적으로 이어 붙인다.
- 뒤집기에도 자주 쓴다.

주요 메서드:

- `append()`
- `reverse()`
- `toString()`

반환값:

- `append()`: `StringBuilder`
- `reverse()`: `StringBuilder`
- `toString()`: `String`

예시:

```java
StringBuilder sb = new StringBuilder();
sb.append("abc");
sb.append(123);
System.out.println(sb.toString());
System.out.println(new StringBuilder("study").reverse().toString());
```

출력:

```text
abc123
yduts
```

언제 쓰는가:

- 반복문에서 문자열을 계속 붙일 때
- 단어 뒤집기
- 출력 누적

### System.out.print / println

역할:

- 출력

차이:

- `print()`: 줄바꿈 없음
- `println()`: 출력 후 줄바꿈

예시:

```java
System.out.print(1);
System.out.print(2);
System.out.println();
System.out.println(3);
```

출력:

```text
12
3
```

## 2. 문자열

### charAt

역할:

메서드:

반환값:

- `char`

```text

u
```

주의:

- 인덱스는 0부터 시작한다.
- 범위를 벗어나면 예외가 난다.

### length

역할:

- 문자열 길이를 구한다.

메서드:

- `length()`

반환값:

- `int`

예시:

```java
String s = "abcde";
System.out.println(s.length());
```

출력:

```text
5
```

### substring

역할:

- 문자열 일부를 잘라낸다.

메서드:

- `substring(int beginIndex)`
- `substring(int beginIndex, int endIndex)`

반환값:

- `String`

예시:

```java
String s = "abcdef";
System.out.println(s.substring(2));
System.out.println(s.substring(1, 4));
```

출력:

```text
cdef
bcd
```

주의:

- `endIndex`는 포함되지 않는다.

### indexOf

역할:

- 특정 문자나 문자열이 처음 나오는 위치를 찾는다.

메서드:

- `indexOf(int ch)`
- `indexOf(String str)`

반환값:

- 찾으면 인덱스 `int`
- 못 찾으면 `-1`

예시:

```java
String s = "banana";
System.out.println(s.indexOf('n'));
System.out.println(s.indexOf("na"));
System.out.println(s.indexOf("zz"));
```

출력:

```text
2
2
-1
```

### toCharArray

역할:

- 문자열을 문자 배열로 바꾼다.

메서드:

- `toCharArray()`

반환값:

- `char[]`

예시:

```java
char[] arr = "abc".toCharArray();
for (char c : arr) {
    System.out.println(c);
}
```

출력:

```text
a
b
c
```

### split

역할:

- 문자열을 정규식 기준으로 나눈다.

메서드:

- `split(String regex)`

반환값:

- `String[]`

예시:

```java
String s = "a,b,c";
String[] arr = s.split(",");
for (String x : arr) {
    System.out.println(x);
}
```

출력:

```text
a
b
c
```

주의:

- 정규식을 받기 때문에 `.` 같은 문자는 그대로 쓰면 의미가 달라진다.
- 단순 공백 분리는 `StringTokenizer`가 더 안전할 때가 많다.

### replace / replaceAll

역할:

- `replace`: 일반 문자열 치환
- `replaceAll`: 정규식 치환

반환값:

- `String`

예시:

```java
String s = "a1b2c3";
System.out.println(s.replace("1", "X"));
System.out.println(s.replaceAll("[0-9]", ""));
```

출력:

```text
aXb2c3
abc
```

주의:

- `replaceAll`은 정규식을 사용한다.

### 대소문자와 문자 판별

## 3. 배열과 정렬

필요한 import:

```java

```

- 배열을 오름차순 정렬한다.

메서드:

- `Arrays.sort(int[] arr)`

## 4. List, Map, Set

### ArrayList

필요한 import:

```java
import java.util.ArrayList;
```

역할:

- 크기가 자동으로 늘어나는 배열

주요 메서드:

- `add(E e)`
- `get(int index)`
- `set(int index, E element)`
- `remove(int index)`
- `size()`

반환값:

- `add`: `boolean`
- `get`: 저장된 원소 타입
- `set`: 이전 값
- `remove`: 삭제된 값
- `size`: `int`

예시:

```java
ArrayList<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);
System.out.println(list.get(0));
System.out.println(list.size());
```

출력:

```text
10
2
```

언제 쓰는가:

- 인접 리스트
- 결과를 순서대로 모을 때

### HashMap

필요한 import:

```java
import java.util.HashMap;
```

역할:

- 키와 값을 저장한다.
- 빠른 조회와 빈도 계산에 강하다.

주요 메서드:

- `put(K key, V value)`
- `get(K key)`
- `getOrDefault(K key, V defaultValue)`
- `containsKey(K key)`
- `remove(K key)`
- `size()`

반환값:

- `put`: 이전 값 또는 `null`
- `get`: 값 또는 `null`
- `getOrDefault`: 값 또는 기본값
- `containsKey`: `boolean`

예시:

```java
HashMap<Character, Integer> map = new HashMap<>();
for (char c : "BACBAA".toCharArray()) {
    map.put(c, map.getOrDefault(c, 0) + 1);
}
System.out.println(map.get('A'));
System.out.println(map.get('B'));
System.out.println(map.containsKey('C'));
```

출력:

```text
3
2
true
```

주의:

- 없는 키에 대해 `get()`을 바로 연산에 쓰면 `NullPointerException`이 날 수 있다.
- 순서가 보장되지 않는다.

## 6. 재귀, 그래프, 자주 쓰는 형태

### 재귀 메서드

역할:

- 함수가 자기 자신을 호출하며 문제를 더 작은 단위로 푼다.

형태:

```java
static void dfs(int level) {
    if (level == n) {
        return;
    }

    dfs(level + 1);
}
```

핵심:

- 종료 조건이 있어야 한다.
- 재귀 깊이가 너무 깊으면 스택 오버플로우가 날 수 있다.

### 방향 배열

역할:

- 2차원 격자에서 상하좌우, 혹은 8방향을 탐색할 때 사용

예시:

```java
int[] dx = {-1, 0, 1, 0};
int[] dy = {0, 1, 0, -1};
```

사용 예:

```java
for (int dir = 0; dir < 4; dir++) {
    int nx = x + dx[dir];
    int ny = y + dy[dir];
}
```

### 인접 리스트

필요한 import:

```java
import java.util.ArrayList;
```

역할:

- 그래프 연결 정보를 저장

예시:

```java
ArrayList<Integer>[] graph = new ArrayList[n + 1];
for (int i = 1; i <= n; i++) {
    graph[i] = new ArrayList<>();
}

graph[1].add(2);
graph[2].add(1);
```

주의:

- 각 칸마다 `new ArrayList<>()`를 꼭 해줘야 한다.

## 7. 정렬 기준 만들기

### Comparable

역할:

- 클래스 내부에 기본 정렬 기준을 정의

메서드:

- `compareTo(T o)`

반환 규칙:

- 음수: 내가 앞
- 0: 같음
- 양수: 내가 뒤

예시:

```java
class Meeting implements Comparable<Meeting> {
    int start;
    int end;

    Meeting(int start, int end) {
        this.start = start;
        this.end = end;
    }

    @Override
    public int compareTo(Meeting o) {
        if (this.end == o.end) return this.start - o.start;
        return this.end - o.end;
    }
}
```

### Comparator 람다

필요한 import:

```java
import java.util.Arrays;
```

역할:

- 정렬할 때 임시 기준을 바로 만든다.

예시:

```java
Arrays.sort(arr, (a, b) -> {
    if (a[0] == b[0]) return a[1] - b[1];
    return a[0] - b[0];
});
```

## 8. 자주 쓰는 자료구조와 메서드의 시간복잡도 감각

이 절은 엄밀한 이론 설명보다, 코딩테스트에서 빠르게 판단하기 위한 감각 정리다.

### 배열

- `arr[i]` 접근: 보통 `O(1)`
- 배열 전체 순회: `O(N)`
- 정렬 `Arrays.sort(arr)`: 보통 `O(N log N)` 감각

예:

```java
int x = arr[3];
```

의미:

- 3번 칸을 바로 꺼내므로 빠르다.

### String

- `length()`: 보통 `O(1)` 감각
- `charAt(i)`: 보통 `O(1)` 감각
- 문자열 전체 순회: `O(N)`

주의:

- 문자열을 `+`로 반복해서 이어 붙이면 느려질 수 있다.
- 반복문 안에서는 `StringBuilder`가 유리하다.

### ArrayList

- 맨 뒤 추가 `add()`: 보통 빠름, 평균 `O(1)` 감각
- `get(i)`: `O(1)`
- 중간 삭제: 뒤 원소들을 당겨야 해서 느릴 수 있음

실전 감각:

- 순서대로 저장하고 꺼내는 용도에 좋다.
- 중간 삽입, 중간 삭제가 많으면 주의한다.

### HashMap

- `put`, `get`, `containsKey`: 평균적으로 `O(1)` 감각

실전 감각:

- 빈도 계산에 매우 강하다.
- 순서를 기대하면 안 된다.

### Queue

- `offer`, `poll`, `peek`: 보통 `O(1)` 감각

실전 감각:

- BFS에 적합하다.

### Deque

- 앞뒤 삽입, 삭제: 보통 `O(1)` 감각

실전 감각:

- 스택, 큐 둘 다 구현 가능하다.

### PriorityQueue

- `offer`, `poll`: 보통 `O(log N)`
- `peek`: 보통 `O(1)` 감각

실전 감각:

- 가장 작은 값, 가장 큰 값을 반복해서 꺼낼 때 좋다.
- 일반 큐보다 내부 정리가 더 필요하므로 조금 더 느리다.

### TreeSet

- 추가, 삭제, 탐색: 보통 `O(log N)`

실전 감각:

- 정렬과 중복 제거가 동시에 필요할 때 쓴다.

### 정렬과 탐색을 같이 볼 때

- 정렬 후 한 번 훑기: 보통 `O(N log N) + O(N)`
- 정렬 후 이분탐색: 보통 `O(N log N) + O(log N)`

실전에서는 큰 항만 남겨서 보통 `O(N log N)`으로 생각한다.

## 9. 코딩테스트에서 자주 나는 실수

- `String` 비교를 `==`로 함
- `substring`의 끝 인덱스를 포함된다고 착각
- `HashMap.get()` 결과가 `null`일 수 있다는 점을 잊음
- `Queue.poll()` 결과가 `null`일 수 있다는 점을 잊음
- `int` 범위를 넘는데도 `long`으로 안 바꿈
- 객체 배열 정렬에서 기준을 잘못 세움
- 방문 체크를 너무 늦게 함
