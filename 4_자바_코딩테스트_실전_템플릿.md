## 1. 사용 원칙

이 문서는 코드를 외우기 위한 문서가 아니라,  
"이 문제에서 어떤 틀을 꺼내야 하는가"를 빠르게 판단하기 위한 문서다.

템플릿을 볼 때는 아래 세 가지를 같이 본다.

1. 언제 쓰는가
2. 어디를 바꿔야 하는가
3. 어떤 실수를 자주 하는가

## 2. 빠른 입력 기본 템플릿

### 언제 쓰는가

- 거의 모든 코딩테스트 문제
- 정수, 문자열, 2차원 배열 입력

### 코드

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.IOException;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        StringTokenizer st = new StringTokenizer(br.readLine());
        int n = Integer.parseInt(st.nextToken());
        int m = Integer.parseInt(st.nextToken());

        int[] arr = new int[n];
        st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            arr[i] = Integer.parseInt(st.nextToken());
        }
    }
}
```

### 바꿔야 하는 부분

- 첫 줄 입력 개수
- 배열 개수
- 숫자 타입 `int` 또는 `long`

### 자주 하는 실수

- 입력 줄이 여러 줄인데 한 줄로만 처리
- 값 범위가 큰데 `int`를 사용

## 3. 2차원 배열 입력

### 언제 쓰는가

- 격자판
- 미로
- 섬 문제
- 시뮬레이션

### 코드

```java
int[][] board = new int[n][m];
for (int i = 0; i < n; i++) {
    StringTokenizer st = new StringTokenizer(br.readLine());
    for (int j = 0; j < m; j++) {
        board[i][j] = Integer.parseInt(st.nextToken());
    }
}
```

### 변형 포인트

- 공백 없는 문자열 입력이면 `charAt`으로 읽는다.

예:

```java
char[][] board = new char[n][m];
for (int i = 0; i < n; i++) {
    String s = br.readLine();
    for (int j = 0; j < m; j++) {
        board[i][j] = s.charAt(j);
    }
}
```

## 4. 방향 배열

### 언제 쓰는가

- 상하좌우 탐색
- 8방향 탐색
- BFS, DFS 좌표 문제

### 코드

```java
int[] dx = {-1, 0, 1, 0};
int[] dy = {0, 1, 0, -1};

for (int dir = 0; dir < 4; dir++) {
    int nx = x + dx[dir];
    int ny = y + dy[dir];

    if (nx < 0 || nx >= n || ny < 0 || ny >= m) continue;
}
```

### 변형 포인트

- 대각선 포함 8방향이면 배열 길이를 8로 바꾼다.

```java
int[] dx = {-1, -1, 0, 1, 1, 1, 0, -1};
int[] dy = {0, 1, 1, 1, 0, -1, -1, -1};
```

### 자주 하는 실수

- `n`, `m` 순서를 바꿈
- `x`, `y`와 `row`, `col` 개념을 혼동

## 5. 문자열 빈도 계산

### 언제 쓰는가

- 문자 개수 세기
- 아나그램
- 구간별 빈도 비교

### 코드

```java
import java.util.HashMap;

HashMap<Character, Integer> map = new HashMap<>();
for (char c : s.toCharArray()) {
    map.put(c, map.getOrDefault(c, 0) + 1);
}
```

### 변형 포인트

- 숫자 빈도면 `HashMap<Integer, Integer>`
- 단어 빈도면 `HashMap<String, Integer>`

### 자주 하는 실수

- `map.get(c) + 1`처럼 없는 키를 바로 연산

## 6. 슬라이딩 윈도우 + HashMap

### 언제 쓰는가

- 길이 `k`인 구간마다 서로 다른 값 개수
- 모든 아나그램 찾기
- 구간 빈도 비교

### 코드

```java
HashMap<Character, Integer> map = new HashMap<>();

for (int i = 0; i < k - 1; i++) {
    char c = s.charAt(i);
    map.put(c, map.getOrDefault(c, 0) + 1);
}

int lt = 0;
for (int rt = k - 1; rt < s.length(); rt++) {
    char add = s.charAt(rt);
    map.put(add, map.getOrDefault(add, 0) + 1);

    // 여기서 현재 구간 검사

    char remove = s.charAt(lt);
    map.put(remove, map.get(remove) - 1);
    if (map.get(remove) == 0) map.remove(remove);
    lt++;
}
```

### 어디를 바꾸는가

- 현재 구간 검사 로직
- 문자 대신 숫자, 문자열 등으로 키 타입 변경

## 7. 스택 템플릿

### 언제 쓰는가

- 올바른 괄호
- 괄호 제거
- 후위식 계산
- 문자열 폭발류

### 코드

```java
import java.util.Deque;
import java.util.ArrayDeque;

Deque<Character> stack = new ArrayDeque<>();

for (char c : str.toCharArray()) {
    if (c == '(') {
        stack.push(c);
    } else {
        if (stack.isEmpty()) {
            System.out.println("NO");
            return;
        }
        stack.pop();
    }
}

System.out.println(stack.isEmpty() ? "YES" : "NO");
```

### 변형 포인트

- 숫자 계산이면 `Deque<Integer>`
- 괄호 종류가 여러 개면 top과 현재 문자를 같이 비교

### 자주 하는 실수

- 마지막에 스택이 비었는지 확인 안 함

## 8. 큐 템플릿

### 언제 쓰는가

- 순서대로 처리
- 시뮬레이션
- BFS

### 코드

```java
import java.util.Queue;
import java.util.LinkedList;

Queue<Integer> queue = new LinkedList<>();
queue.offer(1);
queue.offer(2);

while (!queue.isEmpty()) {
    int x = queue.poll();
}
```

### 자주 하는 실수

- `poll()` 전에 비었는지 확인 안 함

## 9. BFS 기본 템플릿

### 언제 쓰는가

- 최소 이동 횟수
- 정수 상태 공간 탐색
- 시작점에서 목표점까지 최소 단계

### 코드

```java
import java.util.Queue;
import java.util.LinkedList;

Queue<Integer> queue = new LinkedList<>();
boolean[] visited = new boolean[10001];

queue.offer(start);
visited[start] = true;

int level = 0;
while (!queue.isEmpty()) {
    int size = queue.size();
    for (int i = 0; i < size; i++) {
        int cur = queue.poll();

        if (cur == target) {
            System.out.println(level);
            return;
        }

        int[] next = {cur - 1, cur + 1, cur + 5};
        for (int nx : next) {
            if (nx < 1 || nx > 10000) continue;
            if (visited[nx]) continue;
            visited[nx] = true;
            queue.offer(nx);
        }
    }
    level++;
}
```

### 어디를 바꾸는가

- 시작점 `start`
- 목표점 `target`
- 다음 상태 생성 로직
- 범위 체크 조건

### 자주 하는 실수

- 큐에 넣을 때 방문 체크 안 함
- `level` 증가 위치를 잘못 둠

## 10. 좌표 BFS 템플릿

### 언제 쓰는가

- 미로 최단거리
- 섬의 개수
- 토마토류 확산 문제

### 코드

```java
import java.util.Queue;
import java.util.LinkedList;

class Point {
    int x;
    int y;

    Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}

Queue<Point> queue = new LinkedList<>();
boolean[][] visited = new boolean[n][m];

queue.offer(new Point(0, 0));
visited[0][0] = true;

while (!queue.isEmpty()) {
    Point cur = queue.poll();

    for (int dir = 0; dir < 4; dir++) {
        int nx = cur.x + dx[dir];
        int ny = cur.y + dy[dir];

        if (nx < 0 || nx >= n || ny < 0 || ny >= m) continue;
        if (visited[nx][ny]) continue;
        if (board[nx][ny] == 0) continue;

        visited[nx][ny] = true;
        queue.offer(new Point(nx, ny));
    }
}
```

### 변형 포인트

- 거리 배열 `dist[][]` 추가 가능
- 여러 시작점이면 처음에 큐에 전부 넣는다.

## 11. DFS 선택/비선택 템플릿

### 언제 쓰는가

- 부분집합
- 특정 합 만들기
- 바둑이 승차

### 코드

```java
static int[] arr;
static int n;

static void dfs(int level, int sum) {
    if (level == n) {
        return;
    }

    dfs(level + 1, sum + arr[level]);
    dfs(level + 1, sum);
}
```

### 어디를 바꾸는가

- 종료 시 정답 갱신 로직
- 가지치기 조건

### 보강 버전 예시

```java
static void dfs(int level, int sum) {
    if (sum > limit) return;

    if (level == n) {
        answer = Math.max(answer, sum);
        return;
    }

    dfs(level + 1, sum + arr[level]);
    dfs(level + 1, sum);
}
```

## 12. DFS 방문 배열 템플릿

### 언제 쓰는가

- 순열
- 중복 없는 선택
- 방문 여부가 중요한 탐색

### 코드

```java
static int n;
static boolean[] visited;
static int[] pm;

static void dfs(int level) {
    if (level == n) {
        return;
    }

    for (int i = 1; i <= n; i++) {
        if (visited[i]) continue;
        visited[i] = true;
        pm[level] = i;
        dfs(level + 1);
        visited[i] = false;
    }
}
```

### 핵심 포인트

- 들어가기 전에 `visited[i] = true`
- 돌아오면 `visited[i] = false`

## 13. 조합 템플릿

### 언제 쓰는가

- 순서 상관 없이 M개 고르기
- 조합 출력

### 코드

```java
static int n, m;
static int[] comb;

static void dfs(int level, int start) {
    if (level == m) {
        return;
    }

    for (int i = start; i <= n; i++) {
        comb[level] = i;
        dfs(level + 1, i + 1);
    }
}
```

### 자주 하는 실수

- `start`를 안 써서 순열처럼 만들어 버림

## 14. 이진트리 재귀 순회 템플릿

### 언제 쓰는가

- 전위, 중위, 후위 순회
- 트리 재귀 구조 이해

### 코드

```java
static class Node {
    int data;
    Node lt, rt;

    Node(int data) {
        this.data = data;
    }
}

static void dfs(Node root) {
    if (root == null) return;

    // 전위
    System.out.print(root.data + " ");
    dfs(root.lt);

    // 중위 위치
    dfs(root.rt);

    // 후위 위치
}
```

### 변형 포인트

- 출력 위치를 바꾸면 전위, 중위, 후위가 된다.

## 15. 이분탐색 템플릿

### 언제 쓰는가

- 정렬된 배열에서 값 찾기
- 답의 범위를 절반씩 줄이기

### 코드

```java
int lt = 0;
int rt = arr.length - 1;

while (lt <= rt) {
    int mid = (lt + rt) / 2;

    if (arr[mid] == target) {
        System.out.println(mid);
        break;
    } else if (arr[mid] < target) {
        lt = mid + 1;
    } else {
        rt = mid - 1;
    }
}
```

### 전제 조건

- 배열이 정렬되어 있어야 한다.

### 자주 하는 실수

- 정렬을 안 하고 이분탐색 수행
- `lt`, `rt` 갱신 방향 반대로 작성

## 16. 결정 알고리즘 템플릿

### 언제 쓰는가

- 정답의 최솟값, 최댓값을 구하는데
- "이 값으로 가능한가"를 판별할 수 있을 때

예:

- DVD 최소 용량
- 말 사이 최대 거리

### 코드

```java
int lt = minValue;
int rt = maxValue;
int answer = 0;

while (lt <= rt) {
    int mid = (lt + rt) / 2;

    if (canMake(mid)) {
        answer = mid;
        rt = mid - 1;
    } else {
        lt = mid + 1;
    }
}
```

판별 함수 예시:

```java
static boolean canMake(int capacity) {
    int count = 1;
    int sum = 0;

    for (int x : arr) {
        if (sum + x > capacity) {
            count++;
            sum = x;
        } else {
            sum += x;
        }
    }

    return count <= m;
}
```

### 핵심

- `canMake(mid)`가 단조성을 가져야 한다.

## 17. DP 기본 템플릿

### 언제 쓰는가

- 이전 결과를 재사용할 때
- 최대값, 최소값, 경우의 수

### 코드

```java
int[] dp = new int[n + 1];
dp[1] = 1;
dp[2] = 2;

for (int i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}

System.out.println(dp[n]);
```

### 어디를 바꾸는가

- `dp[i]` 정의
- 초기값
- 점화식

### 자주 하는 실수

- `dp[i]` 의미 없이 외워서 씀
- 초기값 누락

## 18. LIS 기본 템플릿

### 언제 쓰는가

- 가장 긴 증가 부분수열

### 코드

```java
int[] dp = new int[n];
Arrays.fill(dp, 1);

for (int i = 0; i < n; i++) {
    for (int j = 0; j < i; j++) {
        if (arr[j] < arr[i]) {
            dp[i] = Math.max(dp[i], dp[j] + 1);
        }
    }
}
```

필요한 import:

```java
import java.util.Arrays;
```

### 핵심

- `dp[i] = i를 마지막 원소로 하는 LIS 길이`

## 19. Union-Find 템플릿

### 언제 쓰는가

- 같은 집합인지 확인
- 연결 여부 판단
- 사이클 검출
- 크루스칼

### 코드

```java
static int[] parent;

static int find(int x) {
    if (parent[x] == x) return x;
    return parent[x] = find(parent[x]);
}

static void union(int a, int b) {
    int fa = find(a);
    int fb = find(b);
    if (fa != fb) parent[fa] = fb;
}
```

초기화:

```java
parent = new int[n + 1];
for (int i = 1; i <= n; i++) {
    parent[i] = i;
}
```

## 20. 우선순위 큐 템플릿

### 언제 쓰는가

- 가장 작은 값 먼저 처리
- 다익스트라

PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offer(5);

## 21. 템플릿 선택표

- 문자열 비교, 변환: 문자열 기본 + `StringBuilder`
- 빈도 계산: `HashMap`
- 연속 구간: 슬라이딩 윈도우
- 최소 이동 횟수: BFS
- 가능한 모든 경우: DFS
- 순열: DFS + 방문 배열
- 조합: DFS + `start`
- 최적화와 경우의 수: DP
- 정렬 후 판단: `Arrays.sort`
- 가능/불가능 판정으로 답 찾기: 결정 알고리즘
- 집합 연결: Union-Find

## 22. 마지막 점검

템플릿을 그대로 복붙하면 끝나는 문제는 많지 않다.  
중요한 것은 아래 세 가지다.

1. 이 템플릿이 왜 맞는지 설명할 수 있어야 한다.
2. 어떤 부분을 문제 조건에 맞게 바꿔야 하는지 알아야 한다.
3. 경계 조건과 예외 처리를 직접 채울 수 있어야 한다.
