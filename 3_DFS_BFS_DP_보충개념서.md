## 1. 세 알고리즘을 구분하는 가장 중요한 질문

문제를 받았을 때 아래 세 질문을 먼저 한다.

1. 가능한 경우를 전부 만들어 봐야 하는가
2. 최소 횟수나 최단 거리를 구하는가
3. 같은 계산이 반복되는가

대체로 다음처럼 연결된다.

- 1번이면 DFS 가능성이 높다.
- 2번이면 BFS 가능성이 높다.
- 3번이면 DP 가능성이 높다.

하지만 실전에서는 섞여 나오는 경우도 많다.  
예를 들어 DFS로 모든 경우를 만들되, 가지치기를 넣거나, DP로 최적화를 추가할 수도 있다.

## 2. DFS

### 정의

현재 선택을 한 뒤, 더 내려갈 수 있을 때까지 한 방향으로 끝까지 내려가 보는 탐색이다.

자료구조 관점에서는 스택 구조다.  
자바에서는 보통 재귀 함수로 구현한다.

### DFS가 잘 맞는 문제

- 부분집합
- 조합
- 순열
- 중복순열
- 모든 경우의 수
- 경로 탐색
- 특정 조건을 만족하는 배치

### DFS를 떠올려야 하는 표현

- 모든 경우
- 가능한 수열
- 부분집합
- 조합
- 경우의 수 출력
- 만들 수 있는 모든 결과

### 쉬운 예시 1: 부분집합

숫자 `[1, 2, 3]`의 부분집합을 모두 구한다고 하자.

각 숫자마다 선택은 2개다.

- 넣는다
- 넣지 않는다

트리로 보면:

```text
1 선택 / 1 비선택
  2 선택 / 2 비선택
    3 선택 / 3 비선택
```

한 가지 가지를 끝까지 내려간 뒤 돌아오는 방식이 DFS다.

### 쉬운 예시 2: 미로 경로 수

격자에서 시작점부터 도착점까지 가는 모든 경로 수를 구한다고 하자.

이 문제는 "몇 개의 경로가 있는가"를 묻는다.  
최단거리나 최소 횟수를 묻는 것이 아니다.  
그래서 BFS보다 DFS가 자연스럽다.

### DFS 설계 순서

DFS 문제를 보면 코드보다 먼저 아래 4가지를 정한다.

1. 상태
2. 종료 조건
3. 다음 선택
4. 복구 필요 여부

예: 순열 문제

- 상태: 현재 몇 번째 자리를 채우는가
- 종료 조건: N개를 다 뽑았는가
- 다음 선택: 아직 사용하지 않은 숫자 중 하나 고르기
- 복구: 방문 배열 원복

### DFS 코드의 기본 골격

```java
static void dfs(int level) {
    if (level == n) {
        return;
    }

    for (int i = 0; i < n; i++) {
        if (visited[i]) continue;
        visited[i] = true;
        dfs(level + 1);
        visited[i] = false;
    }
}
```

### 백트래킹

백트래킹은 DFS를 더 똑똑하게 하는 기법이다.  
"이 아래는 더 봐도 답이 안 나온다"는 순간 바로 되돌아간다.

예:

트럭 최대 적재량이 259인데 현재까지 실은 무게가 이미 300이면  
더 내려가 봐야 의미가 없다.

```java
if (sum > limit) return;
```

이 한 줄이 시간 차이를 크게 만든다.

### 조합과 순열의 차이

초보자가 가장 많이 헷갈리는 부분이다.

#### 순열

순서가 다르면 다른 경우로 본다.

예:

- `1 2`
- `2 1`

서로 다르다.

#### 조합

순서가 달라도 같은 경우로 본다.

예:

- `{1, 2}`
- `{2, 1}`

같은 경우다.

그래서 조합은 보통 `start` 인덱스를 둔다.

```java
static void dfs(int level, int start) {
    if (level == m) return;

    for (int i = start; i <= n; i++) {
        comb[level] = i;
        dfs(level + 1, i + 1);
    }
}
```

### DFS에서 상태를 잘 정하는 법

문제가 막히면 `dfs()` 파라미터부터 다시 본다.

좋은 질문:

- 지금까지 무엇을 선택했는가
- 앞으로 무엇을 선택할 수 있는가
- 정답 판정은 언제 가능한가

예:

- `dfs(level)`
현재 몇 번째 선택까지 왔는가
- `dfs(level, sum)`
현재 선택 횟수와 현재 합
- `dfs(x, y)`
현재 좌표

### DFS의 시간 복잡도 감각

DFS는 경우의 수가 폭발하기 쉽다.

예:

- 부분집합: `2^N`
- 순열: `N!`
- 격자 경로 탐색: 지도 구조에 따라 매우 커질 수 있음

따라서 DFS 문제는 항상:

- 입력 크기
- 중복 상태 여부
- 가지치기 가능성

이 세 가지를 같이 봐야 한다.

### DFS에서 자주 틀리는 부분

- 종료 조건을 늦게 둠
- 방문 배열 복구를 안 함
- 조합 문제를 순열처럼 구현
- 정답 저장 시점을 잘못 잡음
- 가지치기를 안 넣어서 시간 초과

## 3. BFS

### 정의

시작점에서 가까운 상태부터 차례대로 탐색하는 방법이다.

자료구조 관점에서는 큐 구조다.

### BFS가 잘 맞는 문제

- 최소 횟수
- 최단 거리
- 가장 빨리 도달
- 며칠 후 모두 전염
- 레벨 순회

### BFS를 떠올려야 하는 표현

- 최소 몇 번
- 최단거리
- 가장 빠르게
- 며칠 뒤
- 한 번 이동할 때 할 수 있는 행동이 정해져 있음

### 쉬운 예시 1: 숫자 이동

5에서 시작해서 아래 세 가지 이동만 가능하다고 하자.

- `+1`
- `-1`
- `+5`

14까지 최소 몇 번 만에 가는지 묻는다면 BFS를 쓴다.

이유:

- 1번 만에 갈 수 있는 상태를 먼저 모두 본다.
- 그 다음 2번 만에 갈 수 있는 상태를 본다.
- 처음 14를 만나는 순간이 최소 횟수다.

### 쉬운 예시 2: 미로 최단거리

격자에서 한 칸씩 상하좌우 이동할 때  
시작점에서 도착점까지 최소 이동 횟수를 구하는 문제다.

이 역시 가장 가까운 칸부터 퍼져 나가야 하므로 BFS가 맞다.

### BFS의 핵심 구조

```java
Queue<Integer> queue = new LinkedList<>();
boolean[] visited = new boolean[10001];

queue.offer(start);
visited[start] = true;

while (!queue.isEmpty()) {
    int cur = queue.poll();

    for (int next : nextStates(cur)) {
        if (visited[next]) continue;
        visited[next] = true;
        queue.offer(next);
    }
}
```

### 레벨 개념

BFS는 레벨 단위로 생각하면 이해가 쉽다.

- 시작점은 0레벨
- 한 번 이동해서 갈 수 있는 상태는 1레벨
- 두 번 이동해서 갈 수 있는 상태는 2레벨

그래서 최소 횟수 문제는 레벨 수가 답이 된다.

### BFS에서 방문 체크를 언제 해야 하는가

실전에서는 "큐에 넣는 순간" 방문 체크를 하는 것이 안전하다.

이유:

같은 상태가 큐에 여러 번 들어가는 것을 막을 수 있다.

좋은 예:

```java
if (!visited[nx]) {
    visited[nx] = true;
    queue.offer(nx);
}
```

나쁜 예:

큐에서 꺼낼 때 방문 체크를 하면 같은 상태가 중복 삽입될 수 있다.

### 좌표 BFS

2차원 격자에서는 보통 방향 배열을 쓴다.

```java
int[] dx = {-1, 0, 1, 0};
int[] dy = {0, 1, 0, -1};
```

```java
for (int dir = 0; dir < 4; dir++) {
    int nx = x + dx[dir];
    int ny = y + dy[dir];
}
```

### BFS와 DFS의 차이

같은 그래프 탐색이라도 목적이 다르면 선택이 달라진다.

- 경로가 존재하는지만 묻는다
DFS, BFS 둘 다 가능
- 가능한 경로 개수를 센다
DFS가 자연스럽다
- 최단거리, 최소 횟수를 구한다
BFS가 유리하다

### BFS에서 자주 틀리는 부분

- 방문 체크를 늦게 함
- 범위 체크 누락
- 거리 계산 방식을 섞어 씀
- 지나갈 수 없는 칸 조건을 빼먹음
- 시작점 방문 처리 누락

## 4. DP

### 정의

이미 계산한 결과를 저장하고 재사용하는 방법이다.

핵심은 이 문장 하나다.

"같은 계산을 여러 번 하지 않는다."

### DP가 잘 맞는 문제

- 경우의 수
- 최대 점수
- 최소 비용
- 최장 길이
- 일정 규칙이 있는 수열

### DP를 떠올려야 하는 표현

- 방법의 수
- 최대 합
- 최소 비용
- 이전 결과를 이용
- 같은 부분 문제가 반복

### 쉬운 예시 1: 계단 오르기

한 번에 1칸 또는 2칸씩 갈 수 있다고 하자.

`dp[i] = i번 계단에 도달하는 방법의 수`

그러면:

- `dp[1] = 1`
- `dp[2] = 2`
- `dp[3] = dp[2] + dp[1]`

왜냐하면 3번 계단에 오는 마지막 직전 위치는 1번 또는 2번뿐이기 때문이다.

### 쉬운 예시 2: 최대 점수 문제

제한 시간 안에 여러 문제를 풀어 최대 점수를 얻는다고 하자.

이때 `dp[t] = 시간 t 안에 얻을 수 있는 최대 점수`처럼 정의할 수 있다.

그러면 각 문제를 넣을지 말지에 따라 갱신한다.

### DP의 핵심은 상태 정의

초보자는 자꾸 점화식부터 만들려고 한다.  
하지만 더 중요한 것은 `dp[i]`의 의미다.

좋은 상태 정의 예:

- `dp[i] = i번째 수를 마지막으로 하는 최대 증가 부분수열 길이`
- `dp[i] = i번 계단까지 가는 방법의 수`
- `dp[j] = 시간 j 안에 얻는 최대 점수`

나쁜 상태 정의 예:

- `dp[i] = 대충 i까지의 정답`

이렇게 애매하면 점화식도 흔들린다.

### DP 문제 풀이 순서

1. `dp[i]`가 무엇인지 한 문장으로 정의
2. 현재 상태가 어디에서 오는지 생각
3. 점화식 작성
4. 초기값 설정
5. 계산 순서 결정

### 점화식 예시

#### 피보나치

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

#### 계단 오르기

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

#### 최대 증가 부분수열

```text
dp[i] = max(dp[j]) + 1
단, j < i 이고 arr[j] < arr[i]
```

### Top-down과 Bottom-up

#### Top-down

재귀와 메모이제이션을 사용한다.

장점:

- 점화식을 함수 형태로 쓰기 쉽다.

단점:

- 재귀 깊이 부담
- 호출 흐름이 익숙하지 않으면 디버깅이 어렵다.

#### Bottom-up

작은 값부터 반복문으로 쌓아 올린다.

장점:

- 반복문이라 안정적이다.
- 코딩테스트에서는 더 자주 쓰인다.

### DP에서 반복문 방향이 중요한 이유

예를 들어 0/1 배낭 문제는  
같은 문제를 여러 번 쓰면 안 되므로 뒤에서 앞으로 갱신해야 한다.

```java
for (int j = maxTime; j >= time; j--) {
    dp[j] = Math.max(dp[j], dp[j - time] + score);
}
```

반대로 앞으로 갱신하면 같은 문제를 한 번의 라운드에서 여러 번 쓴 것처럼 되어 버릴 수 있다.

### DP와 DFS의 차이

- DFS는 "선택을 만들어 내려가는" 방식
- DP는 "중복 계산을 저장해 재사용하는" 방식

어떤 문제는 두 방식 모두로 보일 수 있다.  
예를 들어 계단 문제를 DFS로 구현하면 경우의 수를 전부 세느라 느릴 수 있지만,  
DP로 바꾸면 중복 계산을 제거할 수 있다.

### DP에서 자주 틀리는 부분

- 상태 정의가 애매함
- 초기값 누락
- 점화식은 맞는데 인덱스가 틀림
- 반복문 방향을 잘못 잡음
- 경우의 수가 커지는데 `int`를 써서 오버플로우

## 5. 세 알고리즘 비교

| 구분 | DFS | BFS | DP |
|---|---|---|---|
| 목적 | 가능한 경우 탐색 | 최소 횟수, 최단 거리 | 중복 계산 제거, 최적화 |
| 대표 자료구조 | 재귀, 스택 | 큐 | 배열 |
| 자주 나오는 표현 | 조합, 순열, 부분집합 | 최소 이동, 최단 거리 | 최대합, 최소비용, 방법의 수 |
| 핵심 질문 | 어떤 선택을 할 것인가 | 어디부터 가까운가 | 이전 결과를 재사용할 수 있는가 |

## 6. 문제를 보고 알고리즘을 고르는 연습

### 예시 1

"숫자들을 골라 합이 목표가 되는 경우가 있는가?"

- 무엇을 고를지 선택해야 한다.
- 가능한 경우를 내려가며 본다.
- DFS

### 예시 2

"5에서 14까지 최소 몇 번 만에 갈 수 있는가?"

- 최소 횟수다.
- 가까운 상태부터 봐야 한다.
- BFS

### 예시 3

"n칸 계단을 오르는 방법 수는?"

- 이전 결과가 반복된다.
- `n-1`, `n-2` 결과를 다시 쓴다.
- DP

### 예시 4

"격자에서 출발점부터 도착점까지 갈 수 있는 모든 경로 수는?"

- 가능한 경로를 전부 세야 한다.
- DFS

### 예시 5

"격자에서 출발점부터 도착점까지 최소 이동 횟수는?"

- 최소 이동 횟수다.
- BFS

## 7. 초보자를 위한 실전 체크리스트

### DFS 체크리스트

- 상태가 무엇인지 정했는가
- 종료 조건이 명확한가
- 방문 배열이나 선택 배열 복구가 필요한가
- 가지치기 조건을 넣을 수 있는가

### BFS 체크리스트

- 시작점은 무엇인가
- 한 번에 이동 가능한 다음 상태는 무엇인가
- 방문 체크를 큐에 넣을 때 하는가
- 범위 체크가 필요한가

### DP 체크리스트

- `dp[i]`를 한 문장으로 정의했는가
- 점화식이 말로 설명되는가
- 초기값이 맞는가
- 반복문 방향이 맞는가

## 8. 추천 학습 방법

1. 같은 문제를 보고 "왜 이 알고리즘인지"를 먼저 말로 설명한다.
2. 코드를 외우기보다 상태 정의와 자료구조를 먼저 적는다.
3. DFS와 BFS는 직접 손으로 작은 예시를 그려 본다.
4. DP는 `dp[i]`의 의미를 문장으로 먼저 쓴 뒤 점화식을 만든다.
5. 틀린 문제는 정답 코드보다 "내가 왜 다른 알고리즘을 떠올렸는지"를 복기한다.
